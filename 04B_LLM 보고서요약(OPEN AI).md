1. 체험판 오픈 일정은 언제인가

```
회의 원문 기준으로는:

- **체험판 오픈 목표일:** **10월 14일**

다만 PM의 발언은 **"체험판 오픈 목표일은 10월 14일입니다."** 이므로, 이는 **확정 일정이 아니라 목표 일정**으로 해석하는 것이 적절합니다.

따라서 회의록 기준 표현은 다음이 정확합니다.

> **체험판 오픈 목표일: 10월 14일**
> 
> ※ 실제 오픈 확정 여부는 회의 원문상 명시되지 않음.

```

---
2. 체험판에서 제외되는 기능은 무엇인가

```
회의 원문 기준으로 **체험판에서 제외되는 기능은 결제 기능**입니다.

**근거**

- 개발: _"결제 기능은 아직 QA가 끝나지 않아서 이번 체험판에는 제외해야 합니다."_
- PM: _"그러면 체험판 범위에는 회원가입, 검색, 저장 기능만 넣겠습니다."_

따라서 체험판 제공 범위는 다음과 같습니다.

|구분|기능|
|---|---|
|포함|회원가입, 검색, 저장|
|제외|결제 기능|

결제 기능은 QA가 완료되지 않아 이번 체험판 범위에 포함되지 않습니다.
```

---
3. 광고 예산 상한은 얼마인가

```
회의 원문 기준으로 **광고 예산 상한은 최대 300만 원**입니다.

**근거**

- 마케팅: _"광고 예산은 최대 300만 원까지만 가능합니다."_

따라서 현재 회의에서 확인된 광고 예산 한도는 **300만 원**이며, 실제 집행 금액은 별도로 결정되지 않았습니다.

```

---
4. 가격은 공개하기로 결정되었는가
```

아니요. **가격 공개 여부는 결정되지 않았습니다.**

**근거**

- PM: _"가격 공개 여부는 아직 결정하지 맙시다. 다음 회의에서 다시 논의할게요."_

따라서 가격 공개는 **미확정 사항**이며, 회의록에서는 **「확인 필요 항목」**으로 분류하는 것이 적절합니다.
```

5. 공개 가능한 개인정보 문구 조건은 무엇인가

```
회의 원문 기준으로 **개인정보 수집 문구는 법무 승인본인 경우에만 외부 공개가 가능합니다.**

**근거**

- 법무: _"개인정보 수집 문구는 법무 승인본으로만 외부 공개 가능합니다."_

따라서 조건은 다음과 같습니다.

|항목|조건|
|---|---|
|개인정보 수집 문구 외부 공개|법무 승인본만 사용 가능|
|법무 미승인 문구|외부 공개 불가|

회의 원문에는 승인 절차, 승인 일정, 승인 주체의 상세 내용은 없으므로 추가 사항은 **확인 필요**입니다.

```
